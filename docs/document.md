## 1. プログラムの使い方

### 1-1. 何をするか
指定した市町村・地区の物件一覧を取得し、**前回からの新規物件（新しい物件ID）** だけを検出して、
その詳細リンクをメールで通知する。価格改定などの「更新」は通知しない。

### 1-2. 監視条件の編集（`config/watchlist.yaml`）
編集するのは基本このファイルだけ。**物件種別(`search_type`)ごとに、監視する地域(市町村)を並べる**。
これで「住宅は南風原だけ／土地は南風原・豊見城・与那原」のように種別ごとに地域を変えられる。
地区(`areas`)を書かなければ（`areas: []`）その市町村全体が対象。

```yaml
watches:
  - search_type: house             # 住宅（新築・中古一戸建て）
    regions:
      - label: 南風原町・住宅（全域）  # 表示用の名前（自由）
        city_slug: haebarucho       # 市町村のローマ字slug
        areas: []                   # 空 = 市町村全体
  - search_type: tochi             # 土地
    regions:
      - label: 南風原町・土地（全域）
        city_slug: haebarucho
        areas: []
      - label: 豊見城市・土地（全域）
        city_slug: tomigusukushi
        areas:
          - takamine               # 地区のローマ字slug（複数可）
      - label: 与那原町・土地（全域）
        city_slug: yonabarucho
        areas: []
```

種別は必要なものだけでよい（土地だけ監視したいなら `house` ブロックを書かない）。
`mansion`（マンション）も同じ要領でブロックを足せる。

**slug（ローマ字）の調べ方**：うちなーらいふで対象地域を開き、アドレスバーのURL末尾を見る。
例 `https://www.e-uchina.net/house/okinawashi/noborikawa`
→ 市町村 `okinawashi` / 地区 `noborikawa`

**メールに出す項目（`mail.detail_fields`）**：各物件のタイトル下に出す項目を選べる（書いた順に表示。不要な行は消せばよい）。
指定可能: `price`(価格) / `madori`(間取り) / `land`(土地面積) / `building`(建物面積) / `built`(築年) /
`address`(住所) / `parking`(駐車場) / `transport`(交通) / `madori_detail`(間取り詳細)

### 1-3. 実行コマンド
プロジェクトフォルダ内で（同梱の仮想環境 `.venv` のPythonを使う）。

#### コマンドプロンプト（cmd.exe）の場合
スタートメニューで「cmd」→ コマンドプロンプトを起動し、まずプロジェクトへ移動する
（パスに空白があるので `"` で囲む）。

```cmd
cd /d "C:\Users\nago yuta\Desktop\claude-pj\jutaku-research"

REM 本番と同じ通常実行（収集 → スナップショット保存 → 差分 → メール送信あり）
.venv\Scripts\python.exe watch.py

REM メールを送らず、新着はログに出すだけ
.venv\Scripts\python.exe watch.py --no-email

REM 保存もメールもしない（収集と差分の確認だけ。お試しに最適）
.venv\Scripts\python.exe watch.py --dry-run
```

#### PowerShell の場合
```powershell
# 本番と同じ通常実行（収集 → スナップショット保存 → 差分 → メール送信あり）
.\.venv\Scripts\python.exe watch.py

# メールを送らず、新着はログに出すだけ
.\.venv\Scripts\python.exe watch.py --no-email

# 保存もメールもしない（収集と差分の確認だけ）
.\.venv\Scripts\python.exe watch.py --dry-run
```

> 「通常実行（メール送信あり）」でメールを送るには `config\mail_password.txt` にGmailアプリパスワード（16文字）が必要。
> 未設定だとメールは自動でスキップされ、新着はログ出力のみになる（設定方法は「2. Gmail送信の設定」を参照）。

タスクスケジューラや手動起動では、上記の通常実行をまとめた **`run.bat`** をダブルクリック/指定すればよい。

### 1-4. 覚えておくこと
- **初回実行はメールを送らない**（その時点の物件を「既知」として記録するだけ）。新着検出は2回目以降。
- 設定に**新しい地域を追加した直後**も、その地域は初回は記録のみ（追加直後の大量メール防止）。
- **新着が無いときも「新着物件なし」メールが届く**（システムが動いている確認になる）。
  ただし頻繁に実行すると毎回届くので、実行間隔は控えめが無難。
- **その日の初回実行時に robots.txt（サイトの巡回ルール）を自動確認する**。
  万一サイトが方針を変えてデータ取得が許可されなくなった場合は、収集・通知を中止し、
  **「自動チェックを停止しました（robots.txt変更）」メールが届く**（1日1通）。
  このメールが来たら新着通知は止まっているので、詳しくは `logs\run.log` を確認する。
- ログは `logs\run.log`（UTF-8）。エラーや新着の履歴はここで確認できる。
- スナップショットは `data\snapshots\` に実行ごとに増え、直近20件（=直近20回ぶん）を残して自動削除される。
  比較は「最新スナップショットと同じ日付の全ファイル」とまとめて行うため、収集の一時的な取りこぼしで
  既知物件が「新着」と誤検知されにくい。

### 1-5. 届くメールの種類（まとめ）
| 件名 | いつ届くか |
|---|---|
| 【うちなーらいふ】新着物件 ○件 | 新着が見つかったとき |
| 【うちなーらいふ】新着物件なし | 新着が無かったとき（初回・新規追加対象を除く） |
| 【うちなーらいふ】自動チェックを停止しました（robots.txt変更） | サイトがデータ取得を許可しなくなったとき（1日1通・要対応） |

※ 初回実行時、および設定に新しい地域を追加した直後は、その対象についてはメールを送らない（記録のみ）。

## 2. Gmail送信の設定（アプリパスワード）

Gmailは通常のログインパスワードではなく、**「アプリパスワード」** を使って送信する。
（アプリパスワードには **2段階認証の有効化が必須**。）

### 2-1. 2段階認証を有効にする
1. https://myaccount.google.com/security を開く
2. 「Googleへのログイン」→「2段階認証プロセス」を **オン** にする（電話番号等で設定）

### 2-2. アプリパスワードのページの入口（重要）
**「アプリ パスワード」ページはメニューからはたどれない。** 一度離れると迷子になりやすいので、
入口は次の **2通りだけ** と覚えておく（2段階認証が「オン」であることが前提）。

1. **直リンクを開く（最も確実・ブックマーク推奨）**
   `https://myaccount.google.com/apppasswords`
2. **検索バーから開く**
   `https://myaccount.google.com` を開く → 画面上部の**検索バーに「アプリ パスワード」**と入力 → 出てきた項目をクリック

> 戻ってもまた来られるよう、上記 1 のURLを**ブックマーク**しておくとよい。

開いた画面で:
1. アプリ名（例: `uchina-watch`）を入力して「作成」
2. 表示される **16桁のパスワード**（例 `abcd efgh ijkl mnop`）を控える
   → 設定時は **空白を除いた16文字**（`abcdefghijklmnop`）を使う

#### 「アプリ パスワード」が見つからない / 「利用できません」と出る場合
主な原因は次のいずれか:
- **2段階認証がまだ完全にオンになっていない** → 先に完全に有効化する（これが一番多い）。
- **会社・組織の Google Workspace アカウント**（例: `＠foresight-sys.co.jp` などの独自ドメイン）で、
  **管理者がアプリパスワードを無効化している** → この場合は個人で取得できない。
  → **送信用に個人の `＠gmail.com` を新規に用意して使う**のが手早い（受信先は普段のアドレスのままでOK）。
  　どうしても会社アカウントで送りたい場合はGoogle Workspace管理者に解放を依頼する。
- 「**高度な保護機能プログラム(Advanced Protection)**」がオンになっている → アプリパスワードは使えない仕様。

### 2-3. パスワードをファイルに保存（今回の方式）
アプリパスワードは設定ファイル(`watchlist.yaml`)には直書きせず、**専用ファイル** に保存する。
本ツールは `config\mail_password.txt` の中身を読み込む（このファイルは `.gitignore` 済みでgitに含まれない）。

手順:
1. 見本 `config\mail_password.txt.example` を同じフォルダにコピーし、名前を **`mail_password.txt`** に変更する
   （または新規に `config\mail_password.txt` を作成）。
2. その中身を、取得した **16文字のアプリパスワードだけ**（1行）にする。例: `abcdefghijklmnop`
   - `#` で始まる行と空行は無視される。空白が混ざっていても自動で除去される（`abcd efgh ijkl mnop` でも可）。
3. 保存して閉じる。これだけで完了（環境変数の設定やターミナル再起動は不要）。

PowerShellで一発で作る例:
```powershell
Set-Content -Path "config\mail_password.txt" -Value "ここに16文字" -Encoding utf8 -NoNewline
```

> 補足: 環境変数 `UCHINA_MAIL_PASS` が設定されている場合は、ファイルより**そちらが優先**される
> （任意のフォールバック）。普段はファイルだけでよい。

⚠️ 注意:
- `mail_password.txt` は平文。**プロジェクトフォルダを他人と共有・クラウド同期・gitに上げない**こと。
  （このPCはOneDrive同期なし＋gitignore済みのため、現状はそのまま運用して問題ない。）

### 2-4. 設定ファイルの送信元・宛先を実アドレスに
`config/watchlist.yaml` の `mail` セクションのアドレスを確認・変更:
```yaml
mail:
  smtp_host: smtp.gmail.com
  smtp_port: 587
  user: 送信用Gmail@gmail.com
  from_addr: 送信用Gmail@gmail.com
  password_file: config/mail_password.txt   # ← 2-3で作ったファイル
  default_to:
    - 通知を受け取りたいアドレス@example.com
```
（現状すでに `user`/`from_addr` は送信用アドレス、`default_to` は受信先が設定済み。）

### 2-5. 動作確認
1. `config\mail_password.txt` に16文字が保存されているか確認する。
2. パスワードファイルが無い/空のままでも、プログラムは送信せず「送信をスキップ」とログに出すだけなので安全。
3. 実際の送信は、2回目以降の実行で**新着が見つかったとき**に行われる。

> うまく送れない時のチェック: 2段階認証がON か / アプリパスワードを正しく保存したか /
> `user` と `from_addr` が送信元Gmailと一致しているか / 会社や回線でSMTP(587)が塞がれていないか。

## 3. Windowsタスクスケジューラへの登録

`run.bat` を定期実行するよう登録する。`run.bat` は `.venv` のPythonをフルパスで呼ぶため、
PATHの状態に依存せず動く。

### 3-1. GUIで登録（基本タスク）
1. スタート →「**タスク スケジューラ**」を検索して開く
2. 右側の「**基本タスクの作成**」をクリック
3. **名前**: 例 `うちなーらいふ新着通知` → 次へ
4. **トリガー**: 「毎日」を選択 → 次へ → 開始時刻（例 8:00）を設定 → 次へ
5. **操作**: 「プログラムの開始」 → 次へ
6. **プログラム/スクリプト** に `run.bat` のフルパスを入力（前後を `"` で囲む）:
   ```
   "C:\Users\nago yuta\Desktop\claude-pj\jutaku-research\run.bat"
   ```
   **「開始（オプション）」** に作業フォルダを入力（`"`で囲まない）:
   ```
   C:\Users\nago yuta\Desktop\claude-pj\jutaku-research
   ```
7. 「完了」

### 3-2. 1日に複数回動かす / 取りこぼし対策（任意だが推奨）
作成したタスクを右クリック →「プロパティ」:
- **［トリガー］タブ** → 編集 →「**詳細設定**」で
  「繰り返し間隔」 = 例 `1時間`、「継続時間」 = `1日` にチェック
  → これで1日のうち1時間ごとに実行される。
- **［設定］タブ** →「**スケジュールされた時刻にタスクを開始できなかった場合、すぐに開始する**」にチェック
  → PCが起動していなかった時間ぶんの取りこぼしを軽減。

### 3-3. ログオフ中も動かしたい場合
- 「プロパティ」→ **［全般］タブ** →「**ユーザーがログオンしているかどうかにかかわらず実行する**」を選択。
- このとき**実行ユーザーは環境変数 `UCHINA_MAIL_PASS` を設定したのと同じユーザー**にすること
  （別ユーザーやSYSTEMだとそのユーザー環境変数が読めず、メール送信できない）。

### 3-4. コマンドで一括登録したい場合（GUIの代わり）
PowerShellで以下を実行（8:00開始・2時間ごと・現在のユーザーで実行）:
```powershell
schtasks /Create /TN "うちなーらいふ新着通知" `
  /TR "\"C:\Users\nago yuta\Desktop\claude-pj\jutaku-research\run.bat\"" `
  /SC HOURLY /MO 2 /ST 08:00 /F
```
削除したい場合:
```powershell
schtasks /Delete /TN "うちなーらいふ新着通知" /F
```

### 3-5. 動作確認
- タスク スケジューラでタスクを右クリック →「**実行**」で即時テスト実行できる。
- 実行後、`logs\run.log` に開始〜終了のログが出ていれば成功。
- 黒い画面（コンソール）が一瞬出るのは正常。

## 困ったときは
- まず `logs\run.log` を確認（エラー内容・新着件数が残る）。
- 物件が0件・エラーが続く場合は、サイト側の内部API仕様変更の可能性あり（[CLAUDE.md](../CLAUDE.md) のAPI仕様を参照）。
- メールが来ない場合は「2. Gmail送信の設定」の動作確認・チェック項目を見直す。
- **「自動チェックを停止しました（robots.txt変更）」メールが来た場合**は、サイトが robots.txt で
  データ取得（`/api/`）を許可しなくなったため、安全のため自動で停止している。
  サイトの方針変更が考えられるので、利用規約・robots.txt（`https://www.e-uchina.net/robots.txt`）を
  確認し、継続の可否を判断する。許可状態に戻れば次の実行から自動で再開する。
